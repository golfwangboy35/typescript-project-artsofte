# Finance Tracker (Angular)

## О проекте

**Finance Tracker** — веб-приложение для учёта личных финансов на **Angular** и **TypeScript**.

Цели продукта:

- отслеживать доходы и расходы;
- просматривать историю транзакций;
- анализировать финансовую статистику;
- управлять категориями расходов и бюджетами;
- ставить финансовые цели и видеть прогресс;
- получать уведомления в реальном времени (превышение бюджета, достижение цели) — на этапе дорожной карты.

Разработка ведётся командой по **Git workflow** и **pull requests**.

## Текущее состояние (MVP)

Реализовано на фронтенде без backend:

- **Вход** (`/login`) и **регистрация** (`/register`) на Reactive Forms с валидацией (email, длина пароля, совпадение паролей).
- **Условная авторизация**: флаг в `localStorage` (`auth=true`); после успешной отправки формы пользователь попадает на главную.
- **`AuthGuard`** защищает основной layout; неавторизованный пользователь перенаправляется на `/login`.
- **Главная панель** (`/`): дашборд с мок-данными (сводка и последние транзакции), боковое меню на **Angular Material** (`MatSidenav`, `MatNavList`), выход из сессии.

Дальше по плану: JWT, реальный API, остальные разделы из таблицы маршрутов ниже.

## Требования и запуск фронтенда

- **Node.js** >= 18 (для Angular CLI 19; для последних версий CLI может потребоваться Node 20+)
- **npm** >= 9

Установка зависимостей и dev-сервер:

```bash
cd web
npm install
npm start
```

Приложение: `http://localhost:4200/`.

Сборка:

```bash
cd web
npm run build
```

Линт:

```bash
cd web
npm run lint
```

Тесты:

```bash
cd web
npm test
```

**PostgreSQL** и прочие сервисы понадобятся позже, когда появится backend.

## Работа с ветками (Git Workflow)

### Ветка `main`

Стабильная версия продукта; прямые коммиты запрещены.

### Ветка `dev`

Интеграция фич после code review.

### Типичный цикл

1. Обновить `dev`: `git checkout dev && git pull`
2. Создать ветку: `git checkout -b feature/имя-задачи`
3. Коммиты, затем `git push origin feature/имя-задачи`
4. Pull Request в **`dev`**
5. После merge — удалить feature-ветку

### Правила

Не коммитить напрямую в `main` и `dev`. Для каждой задачи — отдельная ветка и PR в `dev`.
